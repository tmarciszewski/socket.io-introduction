Namespaces

  • Each namespace emits its own events
  • Clients cannot connect to namespace if it's not "created" on the server
  • You are connecting to namespace so there is no need to rejon after reconnect
  • Each connection is namespaced (default namespace is "/")

Rooms

  • You can join/leave room AFTER you are connected
  • Think about it as filters for received messages
  • You can join many rooms at once
  • You need to rejoin rooms after every reconnect
  • A little "room" is created for every user (with it's ID as name)

Not all messages are created equal - Volatile Messages

  • If it wont hurt you application if some of the messages are dropped
  • When are messages dropped?
    • When network is slow
    • When application is slow
    • When transport is polling and is in the middle of reconnecting
