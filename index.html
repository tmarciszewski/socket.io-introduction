Rooms

  • You can join many rooms at once
  • You need to rejoin rooms after every reconnect
  • You can dynamically join and leave rooms

Namespaces

  • Each connection is namespaced (default namespace is /)
  • Each namespace emits its own "connection" and "disconnect" events
  • Client cannot connect to namespace if it's not "created" on the server
  • You are connecting to namespace so there is no need to rejon after reconnect

Not all messages are created equal

Volatile Messages

  • If it wont hurt you application if some of the messages are dropped
  • When messages are dropped?
    • When network is slow
    • When application is slow
    • When transport is polling and is in the middle of a request-response cycle
